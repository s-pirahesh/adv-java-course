۲-۱: اکوسیستم جاوا و راه‌اندازی محیط

برای شروع ماجراجویی با جاوا، ابتدا باید ابزارهای لازم را بشناسیم و محیط کاری خود را آماده کنیم. اکوسیستم جاوا از سه جزء کلیدی تشکیل شده است که درک نقش هر کدام برای هر توسعه‌دهنده‌ای ضروری است.

اکوسیستم جاوا: JDK، JRE و JVM

این سه جزء اصلی اغلب با هم اشتباه گرفته می‌شوند، اما هر کدام نقش منحصربه‌فرد و مشخصی دارند:

  • کیت توسعه جاوا (JDK - Java Development Kit): این مجموعه ابزار کامل برای توسعه‌دهندگان جاوا است. JDK شامل همه چیزهایی است که برای نوشتن، کامپایل کردن (`javac`) و خطایابی (دیباگ) برنامه‌های جاوا نیاز دارید. برای برنامه‌نویسی به زبان جاوا، حتماً باید JDK روی سیستم شما نصب باشد.
  • محیط اجرای جاوا (JRE - Java Runtime Environment): این مجموعه، حداقل نرم‌افزار مورد نیاز برای اجرای برنامه‌های جاوایی است که قبلاً کامپایل شده‌اند. JRE شامل ماشین مجازی جاوا و کتابخانه‌های اصلی است اما ابزارهای توسعه را ندارد.
  • ماشین مجازی جاوا (JVM - Java Virtual Machine): می‌توان آن را قلب تپنده جاوا دانست. JVM یک "کامپیوتر مجازی" است که بایت‌کد (Bytecode) جاوا را اجرا کرده و شعار معروف جاوا یعنی "یک بار بنویس، همه‌جا اجرا کن" را تحقق می‌بخشد.

نصب JDK

برای شروع برنامه‌نویسی، اولین قدم نصب JDK است. نسخه‌های مختلفی از جاوا وجود دارد که مهم‌ترین آن‌ها نسخه‌های پشتیبانی بلندمدت (LTS - Long-Term Support) مانند نسخه‌های ۸، ۱۱، ۱۷ و ۲۱ هستند. این نسخه‌ها برای محیط‌های تجاری و پروژه‌های بزرگ توصیه می‌شوند. همیشه می‌توانید آخرین نسخه پایدار را نیز نصب کنید.

شما می‌توانید JDK را از وب‌سایت رسمی Oracle یا توزیع‌های دیگر مانند AdoptOpenJDK دانلود کنید. پس از نصب، دستورات `java` و `javac` باید از طریق خط فرمان (Terminal یا CMD) در دسترس باشند.

دانلود آخرین نسخه JDK از سایت Oracle

محیط‌های توسعه یکپارچه (IDE)

اگرچه می‌توان برنامه‌های جاوا را در یک ویرایشگر متن ساده نوشت، اما استفاده از یک IDE فرآیند توسعه را بسیار ساده‌تر و سریع‌تر می‌کند. IDEها امکاناتی مانند تکمیل خودکار کد، خطایابی پیشرفته و مدیریت پروژه را فراهم می‌کنند. برخی از محبوب‌ترین IDEها برای جاوا عبارتند از:

  • IntelliJ IDEA: ساخته شرکت JetBrains، بسیار هوشمند و قدرتمند است و در دو نسخه ارائه می‌شود: Community (رایگان و متن‌باز) و Ultimate (تجاری با امکانات کامل برای توسعه وب و سازمانی).
  • Eclipse: یک IDE کلاسیک، رایگان و متن‌باز که سال‌هاست توسط جامعه بزرگی از توسعه‌دهندگان پشتیبانی می‌شود.
  • Apache NetBeans: پروژه‌ای متن‌باز و رایگان که امکانات کاملی برای توسعه جاوا و سایر زبان‌ها فراهم می‌کند.
  • Oracle JDeveloper: IDE رسمی شرکت اوراکل که تمرکز ویژه‌ای بر توسعه برنامه‌های سازمانی و یکپارچگی با سایر محصولات اوراکل دارد.
  • Visual Studio Code: یک ویرایشگر کد سبک و بسیار محبوب از مایکروسافت که با نصب افزونه‌های مخصوص جاوا (Java Extension Pack)، به یک محیط توسعه قدرتمند تبدیل می‌شود.

همچنین، همانطور که اشاره شد، برای شروع یادگیری و درک بهتر فرآیند کامپایل، می‌توانید از ویرایشگرهای متن ساده مانند Notepad++، Sublime Text یا Vim نیز استفاده کرده و برنامه‌های خود را به صورت دستی از طریق خط فرمان کامپایل و اجرا کنید.

۲-۲: از کد تا اجرا و اولین برنامه !Hello, World

یکی از قدرتمندترین ویژگی‌های جاوا، استقلال از پلتفرم (Platform Independence) است. این یعنی شما کد خود را یک بار می‌نویسید و فایل کامپایل شده آن می‌تواند بدون هیچ تغییری روی ویندوز، مک یا لینوکس اجرا شود. این جادو از طریق بایت‌کد (Bytecode) و JVM انجام می‌شود.

این فرآیند در دو مرحله اصلی صورت می‌گیرد:

  1. کامپایل (Compilation): شما یک فایل متنی با پسوند .java می‌نویسید. کامپایلر جاوا (javac) این کد خوانا برای انسان را به یک زبان میانی به نام بایت‌کد تبدیل کرده و در فایلی با پسوند .class ذخیره می‌کند.
  2. اجرا (Execution): ماشین مجازی جاوا (JVM) فایل .class را گرفته، بایت‌کد را برای سیستم‌عامل و سخت‌افزار مقصد تفسیر و اجرا می‌کند.

یک شروع کلاسیک: !Hello, World

طبق یک سنت قدیمی در برنامه‌نویسی، یادگیری هر زبان جدیدی با نوشتن برنامه‌ای شروع می‌شود که عبارت "!Hello, World" را چاپ کند. این مثال ساده، ساختار اصلی یک برنامه قابل اجرای جاوا را به خوبی نشان می‌دهد.


// File name must be: HelloWorld.java
public class HelloWorld {
    // This is the main entry point of the program
    public static void main(String[] args) {
        // This line prints the text to the console
        System.out.println("Hello, World!");
    }
}

کامپایل و اجرای دستی

برای اجرای این برنامه، ابتدا باید آن را با کامپایلر جاوا به بایت‌کد تبدیل کنیم. فرض کنید فایل بالا را با نام HelloWorld.java ذخیره کرده‌اید. خط فرمان (Terminal یا CMD) را باز کرده و دستورات زیر را وارد کنید:

۱. دستور کامپایل:


javac HelloWorld.java

اگر هیچ خطایی رخ ندهد، این دستور فایلی به نام HelloWorld.class در کنار فایل اصلی شما ایجاد می‌کند. این فایل حاوی بایت‌کد برنامه شماست.

۲. دستور اجرا:


java HelloWorld

توجه کنید که هنگام اجرا، نام کلاس را بدون پسوند .class می‌نویسیم. با اجرای این دستور، JVM برنامه را اجرا کرده و نتیجه زیر را در خروجی نمایش می‌دهد:


Hello, World!

تعامل با ورودی: آرگومان‌های خط فرمان

حالا بیایید برنامه را کمی هوشمندتر کنیم. متد main یک پارامتر به نام String[] args دارد. این پارامتر یک آرایه از رشته‌ها است که مقادیری را که شما هنگام اجرای برنامه در خط فرمان وارد می‌کنید، در خود ذخیره می‌کند. می‌توانیم از این قابلیت برای دریافت ورودی از کاربر استفاده کنیم. 


// File name must be: Greeting.java
public class Greeting {
    public static void main(String[] args) {
        // The 'args' array holds command-line arguments.
        // We check if the array has at least one element.
        if (args.length > 0) {
            // If a name was provided, use the first one.
            String name = args[0];
            System.out.println("Hello, " + name + "!");
        } else {
            // Otherwise, show a generic message.
            System.out.println("Hello, guest! Please provide a name when you run the program.");
        }
    }
}

اجرای برنامه با ورودی و بدون ورودی

ابتدا فایل Greeting.java را کامپایل می‌کنیم: javac Greeting.java. سپس می‌توانیم آن را به دو صورت اجرا کنیم:

۱. اجرا با یک آرگومان ورودی:


java Greeting Sajjad

در اینجا، کلمه "Sajjad" به عنوان اولین عنصر (ایندکس ۰) در آرایه args قرار می‌گیرد. در نتیجه، شرط args.length > 0 درست بوده و خروجی زیر چاپ می‌شود:


Hello, Sajjad!

۲. اجرا بدون آرگومان ورودی:


java Greeting

در این حالت، آرایه args خالی است و طول آن صفر است. بنابراین، برنامه وارد بلوک else شده و خروجی زیر را نمایش می‌دهد:


Hello, guest! Please provide a name when you run the program.

۲-۳: قواعد نام‌گذاری (Naming Conventions)

در جاوا، برای نام‌گذاری متغیرها، متدها، کلاس‌ها و... از شناسه‌ها (Identifiers) استفاده می‌کنیم. این نام‌ها علاوه بر قواعد فنی (مانند شروع نشدن با عدد)، از یک سری قراردادهای نوشتاری پیروی می‌کنند که رعایت آن‌ها برای نوشتن کدهای تمیز، خوانا و حرفه‌ای ضروری است.

قواعد فنی شناسه‌ها

  • شناسه‌ها می‌توانند شامل حروف (a-z, A-Z)، اعداد (0-9)، آندرلاین (_) و علامت دلار ($) باشند.
  • یک شناسه نباید با یک عدد شروع شود.
  • جاوا به بزرگی و کوچکی حروف حساس است (Case-Sensitive). یعنی totalPrice با totalprice دو شناسه متفاوت هستند.
  • نمی‌توان از کلمات کلیدی رزرو شده جاوا (مانند public, class, int) به عنوان شناسه استفاده کرد.

قراردادهای نوشتاری: کلید خوانایی کد

قراردادهای زیر به شما و دیگر توسعه‌دهندگان کمک می‌کنند تا تنها با نگاه کردن به یک نام، نوع و نقش آن را در برنامه حدس بزنید. این موضوع خوانایی و نگهداری کد را به شدت بهبود می‌بخشد.

چگونه این قراردادها به ما کمک می‌کنند؟

وقتی در حال خواندن کد هستید، این سبک‌های نوشتاری مانند یک راهنما عمل می‌کنند:

  • اگر نامی با حرف بزرگ شروع شده باشد (مانند Student یا Greeting)، فوراً متوجه می‌شوید که این یک کلاس (یک نقشه یا الگو) است.
  • اگر نامی با حرف کوچک شروع شده باشد (مانند studentName یا calculateAvg)، می‌دانید که با یک متغیر یا یک متد سروکار دارید. حالا چگونه این دو را از هم تشخیص دهیم؟
    • اگر بعد از نام، پرانتز () وجود داشته باشد، قطعاً یک متد است: calculateAvg().
    • اگر پرانتزی وجود نداشته باشد، یک متغیر (یا فیلد) است: studentName.
  • اگر نامی با حروف بزرگ و آندرلاین نوشته شده باشد (مانند MAX_SIZE)، فوراً می‌فهمید که این یک مقدار ثابت است که مقدار آن در طول برنامه تغییر نخواهد کرد.

رعایت این قراردادها نه تنها کد شما را برای دیگران خواناتر می‌کند، بلکه به خود شما نیز کمک می‌کند تا با سرعت و دقت بیشتری کد خود را توسعه داده و خطایابی کنید.

۲-۴: آناتومی یک فایل جاوا

هر فایل با پسوند .java یک واحد کامپایل در زبان جاوا است و ساختار مشخصی دارد. این ساختار می‌تواند از سه بخش اصلی و به ترتیب زیر تشکیل شود. البته فقط بخش سوم (تعریف کلاس) اجباری است.

بخش ۱: `package`

این دستور (در صورت وجود) باید اولین خط کد در فایل باشد و مشخص می‌کند که تمام کلاس‌های تعریف شده در این فایل، به کدام بسته تعلق دارند. در مورد پکیج‌ها در بخش بعدی بیشتر صحبت خواهیم کرد.

بخش ۲: `import`

برای استفاده از کلاسی که در یک پکیج دیگر قرار دارد، باید آن را به فایل خود "وارد" یا import کنیم. این کار به کامپایلر می‌گوید که کلاس مورد نظر را از کجا پیدا کند.

  • وارد کردن یک کلاس مشخص: برای این کار، از نام کامل کلاس (Fully Qualified Name) استفاده می‌کنیم که از ترکیب نام بسته + . + نام کلاس تشکیل شده است. 
    import java.util.Scanner; // فقط کلاس Scanner را وارد می‌کند
  • وارد کردن تمام کلاس‌های یک پکیج: گاهی نیاز به استفاده از چندین کلاس از یک پکیج داریم. در این حالت، می‌توانیم از علامت ستاره (*) یا Wildcard استفاده کنیم تا تمام کلاس‌های عمومی (public) آن پکیج را یکجا وارد کنیم. 
    import java.time.*; // تمام کلاس‌های عمومی پکیج time را وارد می‌کند

    نکته: این کار باعث کندی برنامه نمی‌شود، بلکه فقط به کامپایلر اجازه دسترسی به آن کلاس‌ها را می‌دهد.

  • پکیج `java.lang`: این پکیج شامل کلاس‌های بنیادی و بسیار پرکاربرد جاوا مانند String، System، Integer و Object است. به دلیل اهمیت بالای این پکیج، جاوا آن را به صورت ضمنی (Implicitly) برای شما در تمام فایل‌ها import می‌کند. به همین دلیل ما در مثال `HelloWorld` نیازی به نوشتن import java.lang.System; نداشتیم.

بخش ۳: تعریف کلاس‌ها

قلب هر فایل جاوا، تعریف کلاس‌های آن است. در این مورد دو قانون بسیار مهم وجود دارد:

  1. یک فایل .java می‌تواند شامل هر تعداد کلاس غیرعمومی (non-public) باشد. این کلاس‌ها معمولاً کلاس‌های کمکی هستند که فقط در همان فایل استفاده می‌شوند.
  2. در هر فایل .java، حداکثر یک کلاس می‌تواند از نوع public باشد.

قانون طلایی جاوا: تناظر کلاس Public و نام فایل

اگر کلاسی را به صورت public تعریف کنید، نام آن کلاس باید دقیقاً با نام فایل (بدون پسوند .java) یکسان باشد. این یک قانون اجباری است و در صورت عدم رعایت، کد شما کامپایل نخواهد شد. این قانون به JVM کمک می‌کند تا کلاس‌های عمومی را به سرعت و بدون ابهام در پروژه پیدا کند.

برای مثال، اگر کدی حاوی public class Student { ... } باشد، باید حتماً در فایلی به نام Student.java ذخیره شود.

۲-۵: مدیریت پروژه با پکیج‌ها

وقتی تعداد فایل‌ها و کلاس‌ها در یک پروژه زیاد می‌شود، مدیریت آن‌ها بدون یک ساختار منظم تقریباً غیرممکن است. پکیج‌ها در جاوا، مکانیزم اصلی برای سازماندهی کلاس‌ها در گروه‌های منطقی و مدیریت پیچیدگی پروژه‌های بزرگ هستند.

چرا به پکیج‌بندی نیاز داریم؟ استفاده مجدد از نرم‌افزار

یکی از اهداف اصلی مهندسی نرم‌افزار، **استفاده مجدد از کد (Code Reusability)** است. ما نمی‌خواهیم چرخ را از اول اختراع کنیم. اگر یک تیم کلاسی برای کار با تاریخ شمسی نوشته است، تیم‌های دیگر نیز باید بتوانند به سادگی از آن استفاده کنند.

مفهوم کارخانه نرم‌افزار (Software Factory)

پکیج‌ها به ما این امکان را می‌دهند که به تدریج یک "کارخانه نرم‌افزار" برای خود یا سازمانمان بسازیم. در این دیدگاه، هر پکیج مانند یک قطعه استاندارد و تست‌شده در یک کارخانه است (مثلاً پکیج کار با دیتابیس، پکیج محاسبات ریاضی، پکیج تاریخ شمسی). برای ساخت یک برنامه جدید، به جای شروع از صفر، این قطعات آماده را از قفسه برداشته و با کنار هم قرار دادن آن‌ها، محصول نهایی را با سرعت، کیفیت و هزینه بسیار کمتری تولید می‌کنیم.

قانون طلایی پکیج‌ها: تناظر پکیج و پوشه

جاوا برای پیدا کردن کلاس‌ها، یک قانون ساده و سخت‌گیرانه دارد: ساختار پکیج در کد شما (package com.pirahesh.utils;) باید دقیقاً با ساختار پوشه‌ها در پروژه شما مطابقت داشته باشد. یعنی فایل‌های این پکیج باید در مسیر فیزیکی .../com/pirahesh/utils/ قرار داشته باشند.

قرارداد نام‌گذاری پکیج‌ها

برای جلوگیری از تداخل نام پکیج‌ها در سطح جهانی (مثلاً دو شرکت مختلف پکیجی به نام utils بسازند)، یک قرارداد جهانی وجود دارد: از نام دامنه اینترنتی معکوس به عنوان پیشوند نام پکیج استفاده کنید.

  • اگر دامنه شما pirahesh.com است، نام پکیج‌های شما با com.pirahesh شروع می‌شود (مانند: com.pirahesh.utils, com.pirahesh.models).

این قرارداد تضمین می‌کند که نام پکیج شما در سطح جهانی یکتا باشد، زیرا هیچ کس دیگری نمی‌تواند از دامنه شما استفاده کند.
