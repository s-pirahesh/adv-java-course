اگر شخصی از شما بپرسد: "قدرتمندترین ویژگی برنامه‌نویسی شیءگرا چیست؟"، پاسخ یک کلمه است: Polymorphism (چندریختی). این مفهوم، ترکیب Inheritance، Override و Upcasting است و به ما اجازه می‌دهد کدهایی بنویسیم که انعطاف‌پذیر، قابل توسعه و قابل استفاده مجدد هستند.

۱. معنای چندریختی

معنای لغوی:

  • Poly = چند (Many)
  • Morph = شکل (Form)
  • Polymorphism = داشتن چند شکل

معنای برنامه‌نویسی:

یک رفرنس از نوع والد می‌تواند به اشیاء از انواع مختلف فرزندان اشاره کند، و در زمان اجرا، رفتار مناسب هر شیء خاص اجرا شود.

به زبان ساده: چندریختی به ما اجازه می‌دهد یک کد عمومی بنویسیم که با انواع مختلف اشیاء کار کند، و هر شیء رفتار مخصوص خودش را اجرا کند.

۲. مشکل بدون چندریختی

فرض کنید می‌خواهیم حیوانات مختلف در یک باغ وحش را غذا بدهیم. بدون چندریختی، کد ما اینگونه خواهد بود:

public class ZooKeeper {
    public void feedDog(Dog dog) {
        dog.eat();
    }

    public void feedCat(Cat cat) {
        cat.eat();
    }

    public void feedBird(Bird bird) {
        bird.eat();
    }

    // اگر حیوان جدید اضافه شود، باید متد جدید بنویسیم!
    public void feedLion(Lion lion) {
        lion.eat();
    }
}
مشکلات این روش:
  • کد تکراری زیاد.
  • عدم انعطاف‌پذیری (برای هر نوع جدید باید کلاس ZooKeeper تغییر کند).
  • نقض اصل Open/Closed.

قدرت چندریختی

حالا همین کد را با چندریختی می‌نویسیم:

public class ZooKeeper {
    // یک متد برای همه حیوانات!
    public void feedAnimal(Animal animal) {
        animal.eat();  // هر حیوان به روش خودش غذا می‌خورد
    }
}

// استفاده:
ZooKeeper keeper = new ZooKeeper();
keeper.feedAnimal(new Dog());   // Upcasting
keeper.feedAnimal(new Lion());  // Upcasting - بدون تغییر در ZooKeeper!

۵. مثال کامل: سیستم پرداخت

این مثال ترکیبی از Inheritance، Override و Upcasting را نشان می‌دهد.

// 1. کلاس پایه
abstract class Payment {
    protected double amount;

    public Payment(double amount) { this.amount = amount; }

    // 2. متد انتزاعی برای Override
    public abstract void process();
}

// فرزند 1
class CreditCardPayment extends Payment {
    public CreditCardPayment(double amount) { super(amount); }

    @Override
    public void process() {
        System.out.println("Processing Credit Card: $" + amount);
    }
}

// فرزند 2
class PayPalPayment extends Payment {
    public PayPalPayment(double amount) { super(amount); }

    @Override
    public void process() {
        System.out.println("Processing PayPal: $" + amount);
    }
}

// 3. استفاده از چندریختی
class PaymentProcessor {
    // این متد با "هر نوع" پرداختی کار می‌کند
    public void processPayment(Payment payment) {
        payment.process(); // متد مناسب در Runtime اجرا می‌شود
    }
}

// اجرا
public class Main {
    public static void main(String[] args) {
        PaymentProcessor proc = new PaymentProcessor();

        Payment p1 = new CreditCardPayment(100);
        Payment p2 = new PayPalPayment(50);

        proc.processPayment(p1); // خروجی: Processing Credit Card...
        proc.processPayment(p2); // خروجی: Processing PayPal...
    }
}

۶. Dynamic Method Dispatch (ارسال پویای متد)

وقتی کد payment.process() را می‌نویسیم، JVM چگونه تصمیم می‌گیرد کدام متد را فراخوانی کند؟

۷. مثال واقعی: برنامه رسم اشکال

abstract class Shape {
    public abstract void draw();
    public abstract double calculateArea();
}

class Circle extends Shape {
    private double radius;
    public Circle(double r) { this.radius = r; }

    @Override
    public void draw() { System.out.println("Drawing Circle"); }
    @Override
    public double calculateArea() { return Math.PI * radius * radius; }
}

class Rectangle extends Shape {
    private double w, h;
    public Rectangle(double w, double h) { this.w = w; this.h = h; }

    @Override
    public void draw() { System.out.println("Drawing Rectangle"); }
    @Override
    public double calculateArea() { return w * h; }
}

public class DrawingApp {
    public static void main(String[] args) {
        // آرایه چندریختی
        Shape[] shapes = new Shape[2];
        shapes[0] = new Circle(5);
        shapes[1] = new Rectangle(4, 6);

        for (Shape s : shapes) {
            s.draw(); // هر شکل خودش را می‌کشد
            System.out.println("Area: " + s.calculateArea());
        }
    }
}

۸. مزایای چندریختی

مزیت توضیح
کد قابل استفاده مجدد یک متد برای انواع مختلف (مثل processPayment).
انعطاف‌پذیری افزودن نوع جدید بدون تغییر کد قدیم.
کد خوانا منطق واضح و بدون پیچیدگی (حذف if-else).
اصل Open/Closed باز برای توسعه، بسته برای تغییر.
کاهش وابستگی کد به جزئیات وابسته نیست، بلکه به کلاس انتزاعی وابسته است.

۹. چندریختی در مقابل Type Checking

یک اشتباه رایج، استفاده از instanceof به جای چندریختی است.

❌ روش بد (بدون چندریختی):

public void processShape(Shape shape) {
    if (shape instanceof Circle) {
        Circle c = (Circle) shape;
        System.out.println("Processing circle...");
    } else if (shape instanceof Rectangle) {
        Rectangle r = (Rectangle) shape;
        System.out.println("Processing rectangle...");
    }
    // مشکل: هر شکل جدید = یک if-else جدید!
}

✅ روش خوب (با چندریختی):

public void processShape(Shape shape) {
    shape.draw();  // هر شکل خودش را می‌کشد
}

۱۰. اشتباهات رایج

الف) فراموش کردن @Override

class Dog extends Animal {
    // ❌ اشتباه تایپی، متد جدید ساخته می‌شود نه Override
    public void makeSoung() {
        System.out.println("Woof");
    }
}
// animal.makeSound() متد والد را اجرا می‌کند!

ب) Downcasting غیرضروری

// ❌ بد
if (animal instanceof Dog) {
    Dog dog = (Dog) animal;
    dog.makeSound();
}

// ✅ خوب
animal.makeSound();

۱۱. تمرین‌های عملی

تمرین 1: سیستم حیوانات

کلاس انتزاعی Animal با متدهای makeSound() و move() بسازید. سه کلاس فرزند Dog, Cat, Bird با پیاده‌سازی‌های متفاوت ایجاد کنید. در main، آرایه‌ای از Animal بسازید و چندریختی را نمایش دهید.

تمرین 2: سیستم حقوق کارمند

کلاس انتزاعی Employee با متد calculateSalary() بسازید. دو فرزند:

  • HourlyEmployee: حقوق = ساعت کار × نرخ ساعتی
  • SalariedEmployee: حقوق = حقوق ماهانه ثابت

متدی بنویسید که Employee[] بگیرد و مجموع حقوق را محاسبه کند.

تمرین 3: بازی دشمنان

کلاس Enemy با متدهای attack() و takeDamage(int amount) بسازید. سه نوع دشمن:

  • Goblin: حمله سریع، آسیب کم
  • Orc: حمله آهسته، آسیب زیاد
  • Dragon: حمله آتشین، پرواز

کلاس Game بسازید که آرایه‌ای از دشمنان را مدیریت کند.

تمرین 4: پیش‌بینی خروجی

کد زیر چه خروجی می‌دهد و چرا؟

class A {
    public void m1() { System.out.print("A1 "); }
    public void m2() { System.out.print("A2 "); }
}
class B extends A {
    @Override
    public void m1() { System.out.print("B1 "); }
}
class C extends B {
    @Override
    public void m1() { System.out.print("C1 "); }
    @Override
    public void m2() { System.out.print("C2 "); }
}

A obj1 = new B();
obj1.m1(); obj1.m2();

A obj2 = new C();
obj2.m1(); obj2.m2();

تمرین 5: سیستم نوتیفیکیشن

کلاس انتزاعی Notification با متد send(String message) بسازید. سه نوع:

  • EmailNotification
  • SMSNotification
  • PushNotification

کلاس NotificationService با متد sendToAll(Notification[], String) بسازید که پیام را به همه روش‌ها ارسال کند.

خلاصه بخش چهاردهم

  • ✅ Polymorphism: یک رفرنس، چند رفتار.
  • ✅ سه شرط اصلی: Inheritance + Override + Upcasting.
  • ✅ Dynamic Method Dispatch: JVM در Runtime متد را انتخاب می‌کند.
  • ✅ مزیت اصلی: کد قابل توسعه، انعطاف‌پذیر و قابل استفاده مجدد.
  • ✅ اصل Open/Closed: باز برای توسعه، بسته برای تغییر.
  • ✅ اجتناب از instanceof: استفاده از Override به جای Type Checking.
